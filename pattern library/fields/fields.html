Input Fields

Notes

Text inputs are paired with a corresponding label element, indicating what kind of input is required. There is only a bottom-border property for input fields, with a :focus peudo-class following the brand colours.

When an input field is required, using the required class on the label will prompt an ::after pseudoi-class, displaying the *Required label.

Notes

Text inputs with buttons are paired with a corresponding label element to indicate the purpose of the input field. To align the input and button inline, a display: flex style is applied to the parent container with the class input-with-button. This ensures that the input and button appear on the same line.

Clicking the button triggers a :active style, causing a color change to indicate the button's active state.

Please enter a valid email address.

Notes

Text inputs with help information are designed to provide additional guidance to users. A label element is used to describe the input field's purpose. Help information is displayed in a .help-text element beneath the input field. When the input field receives focus, the help text eases into view using a transition effect.

This transition effect enhances the user experience by smoothly revealing the help information when needed.

Password must be at least 8 characters.

Notes

Validation error text inputs are used to indicate input errors on required fields. A label element is employed to convey the input field's importance. When an error occurs, an .error class is added to the input field to trigger the error state. The error message is displayed in the .error-message element beneath the input field using our --colourRedError colour. The bottom-border of the input field is also styled in this colour.

For demonstration purposes, the error state can be simulated by manually adding the .error class to the input field. In practice, this class is typically applied dynamically when validation errors occur.
